FITFORGE · §19 · NON-HAPPY PATTERNS

空、載入、出錯 — 三種真實狀態

90% 的設計時間在快樂路徑、10% 在這裡。但使用者在這 10% 形成你 90% 的印象。 空狀態要邀請、載入要誠實、錯誤要可救援。

11 個 pattern · Light + Dark 插圖 · 線稿 · primary 單色 · 200×200 Loading · skeleton 優先 · spinner 最多 2 秒 本頁所有 skeleton / spinner 為真實動畫

§19.4 · 規則總覽

文案三要
  • · 標題講事實（還沒有 / 沒有符合 / 找不到）、不哀求
  • · 副標講為什麼或下一步是什麼
  • · CTA 用動詞 + 受詞（「開始第一次訓練」勝過「開始」）
  • · 不寫「Oops」「糟糕」「您好遺憾」
  • · 錯誤碼放底部 mono、給工程查、不嚇使用者
Skeleton 與 Spinner
  • · skeleton 形狀必須對應真實內容、最後一個 row 用 60% opacity 暗示「往下還有」
  • · shimmer 1.6s · linear · 由左向右
  • · spinner 預期 < 2s、超過 2s 換成 skeleton + 進度提示
  • · 永遠尊重 prefers-reduced-motion · 停止 shimmer 與 rotation
  • · 切換 loading → loaded 用 200ms fade、不 pop
錯誤救援階梯
  • · 第 1 步：明顯 CTA（重試 / 重新整理 / 回首頁）
  • · 第 2 步：text button「回報問題」/「先離線繼續」
  • · 第 3 步：底部 mono 錯誤碼 · 截圖可貼工程
  • · 永遠提供「離線繼續」當第二路 — FitForge 訓練不該被網路綁架
