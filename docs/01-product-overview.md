# 01 — 產品概述 (Product Overview)

> 本檔回答「**為什麼**做這個 app、做給誰、做到哪裡」。技術讀者也務必先讀，否則後面的架構決策會看起來像憑空冒出。

---

## 1. 一句話定位 (Positioning Statement)

> 給「想開始重訓但不知從何下手」的新手，一個**離線可用、零註冊、教學完整**的健身入門 PWA — 三套預設課表帶你入門、Lottie 動畫教你動作、簡潔紀錄追蹤進步。

---

## 2. 為什麼是這個產品

### 2.1 觀察到的問題 (Problem Validation)

健身新手 (0–3 個月訓練史) 第一次走進健身房時，遭遇的痛點：

| 痛點                                                | 既有解法                          | 既有解法的問題                                    |
| --------------------------------------------------- | --------------------------------- | ------------------------------------------------- |
| 不知道做什麼動作                                    | YouTube 教學                      | 訊息過載、缺乏結構、教練商業利益偏見              |
| 不知道課表怎麼排                                    | 健身房教練 / 訂閱 app             | 教練貴 + 偏見、訂閱 app 強制註冊 + 大量無關功能   |
| 訓練中不知道做幾組幾下、休息多久                    | 紙筆 / Notes app                  | 笨重、無倒數計時、無動作圖示                      |
| 訓練中網路不穩                                      | 雲端 app                          | 操作卡頓、資料遺失風險                            |
| 隱私顧慮 (體重、訓練習慣不想被廠商蒐集)              | -                                 | 多數 app 預設上傳、隱私聲明複雜                   |

### 2.2 V1 為什麼鎖定「離線、零註冊、新手」

- **離線**：訓練場景 — 地下室健身房訊號差、訓練中專注度需要、本地操作 < 50ms 體驗碾壓網路 round-trip。
- **零註冊**：新手心理門檻 — 還沒確定要不要持續就要先給 email、密碼，等於趕客。
- **新手**：市場最大、競爭最激烈、痛點最具體；進階訓練者已有偏好的工具 (Strong、Hevy)、難以撼動。

### 2.3 既有競品快速掃描

| App         | 定位         | 商業模式      | V1 對比差異點                                    |
| ----------- | ------------ | ------------- | ------------------------------------------------ |
| Strong      | 紀錄為主     | Freemium      | 我們強教學、預設課表完整、零註冊                  |
| Hevy        | 紀錄 + 社群  | Freemium      | 同上、且 V1 無社群更聚焦                          |
| Nike Training| 內容為主    | 免費 + 帳號   | 我們是純訓練、無有氧課表干擾、隱私零                |
| Fitbod      | AI 排課表    | 訂閱          | V1 不做 AI、聚焦核心；V2 走差異化                 |
| 健身房紙本  | 經典        | 免費          | 我們是它的數位版 + 動作圖 + 倒數                  |

**差異化 = 入門完整度 × 隱私 × 離線**。

## 4. MVP 範疇 (In-Scope)

### 4.1 核心三件套

#### 4.1.1 預設課表 + 自訂課表
- **內建 3 套課表** (詳見 [05-domain-logic.md](./05-domain-logic.md) §3)：
  - Beginner Full Body 全身入門 (一週 3 次、每次 45 分)
  - Upper / Lower 上下分化 (一週 4 次、每次 45 分)
  - Push / Pull / Legs 推拉腿 (一週 3–6 次、每次 60 分)
- **自訂課表編輯器**：用戶可從動作庫拖拉建立、設定組數/次數/休息時間
- **單次套用**：今天就用這套課表 (不需訂閱、不卡日曆功能)

#### 4.1.2 動作圖庫
- **MVP 30 個動作** (詳見 [04-data-model.md](./04-data-model.md) §5)：涵蓋三大健身房常見動作 (Squat、Bench、Deadlift、Row、OHP 等) + 啞鈴常用動作 + 自體重 fallback
- **每個動作含**：Lottie 動畫、繁中名稱、英文名稱、目標肌群、難度標籤、文字步驟、3 點要注意事項、常見錯誤
- **搜尋 + 篩選**：依肌群、器材、難度

#### 4.1.3 訓練紀錄
- **訓練中 Logger**：當下記每組重量 × 次數 × 自感強度 (RPE 1–10，選填)
- **組間倒數**：依課表預設秒數，可手動調整、可跳過
- **完成總結**：本次訓練時長、總組數、總噸位 (簡單統計、不做曲線)
- **歷史列表**：日期排序的訓練紀錄、點入看單次詳情

### 4.2 輔助功能

- **Onboarding 流程** (3 步驟、可跳過)：訓練目標 → 訓練頻率 → 可用器材
- **設定**：單位切換 (kg / lb)、主題 (淺色 / 深色 / 跟系統)、資料匯出 (JSON 下載)、資料清除
- **PWA 安裝引導**：首次訪問第 3 次顯示安裝提示

### 4.3 非功能性必備

- **完整離線**：第一次載入後可斷網使用
- **可安裝**：Web App Manifest 完整、可加到主畫面
- **效能**：FCP < 1.5s (4G)、TTI < 2.5s (4G)、Lighthouse PWA / Perf 全 ≥ 90
- **可訪問性**：基本 a11y (語義 HTML、鍵盤可操作、screen reader friendly)
- **i18n 預備**：所有文案抽 key，V1 只有 `zh-TW` locale，但結構就位

## 5. 非範疇 (Out-of-Scope for V1)

> 重要：每一項都標「為什麼不做」，避免未來爭執。

| 項目                  | 為什麼 V1 不做                                                                                             | 何時做       |
| --------------------- | ---------------------------------------------------------------------------------------------------------- | ------------ |
| 註冊 / 帳號 / OAuth   | V1 無雲端，無需身份識別                                                                                    | V2 雲端同步時 |
| 雲端同步              | V1 個人裝置內就足以驗證 UX；雲端引入後端成本與隱私複雜度                                                    | V2           |
| AI 個人教練           | 沒有訓練資料時，AI 建議無從 grounding；V1 先累積資料                                                       | V2           |
| 飲食 / 卡路里         | 資料模型大、第三方 API 成本、UX 完全不同的領域                                                              | V2 或永不    |
| 穿戴整合              | 需要原生 capability (HealthKit / Health Connect)，V1 是 PWA                                                | V2 RN 版時   |
| 社群 / 好友 / 貼文    | 引入後端 + moderation + 內容政策；V1 焦點是「能不能練起來」、不是「能不能 po 文」                            | V2 後段甚至 V3 |
| 體態照 / 體重曲線     | 量化進度模組獨立、V1 聚焦訓練本身                                                                          | V2           |
| 金流 / 訂閱           | V1 是免費個人專案                                                                                          | 看商業化規劃 |
| 多國語系              | i18n 結構預備、V1 只接 zh-TW                                                                                | V2           |
| Native App Store      | PWA 可安裝、跳過 Apple/Google 審核摩擦；V2 RN 時再考慮                                                      | V2 或永不    |
| Apple Watch / Wear OS | 需要伴侶 native app                                                                                        | V2           |
| 即時心率              | 需要藍牙 / 穿戴                                                                                            | V2 或永不    |
| 教練端 (B2B2C)        | V1 用戶為新手、不是教練；架構雖支援多用戶但 UI 不會做教練端                                                  | V3           |
| 影片教學 (取代 Lottie) | 製作成本高、檔案大、離線壓力大                                                                              | V2 可選      |
| 進度照前後對比        | 量化進度模組                                                                                                | V2           |
| 訓練計時器之外的工具   | 例：1RM 計算器、TDEE、BMI — 都是 nice-to-have、不是核心                                                    | V2 看需求    |

## 6. 使用者故事 (User Stories — MVP)

> 採 INVEST 標準、Given/When/Then 格式。

### Epic 1: 首次體驗

- **US-101**：As 新手，I want 第一次打開 app 就被引導到適合我的課表，so that 我不用自己研究。
  - Given 我是首次訪問，When 完成 3 步 onboarding 並按「開始第一次訓練」，Then 我看到符合我目標 + 頻率 + 器材的預設課表第一日內容。
- **US-102**：As 新手，I want 跳過 onboarding 直接進首頁，so that 我可以先逛逛。
- **US-103**：As 新手，I want 把 app 加到手機主畫面，so that 我不用每次找瀏覽器書籤。

### Epic 2: 學動作

- **US-201**：As 新手，I want 看到每個動作的 Lottie 動畫示範，so that 我知道動作怎麼做。
- **US-202**：As 新手，I want 看到動作的「3 點要注意」，so that 我不要受傷。
- **US-203**：As 新手，I want 依肌群篩選動作，so that 我可以練我想練的部位。
- **US-204**：As 新手，I want 搜尋動作中文或英文名稱，so that 我快速找到。

### Epic 3: 開始訓練

- **US-301**：As 新手，I want 從首頁一鍵開始今天的訓練，so that 不要點來點去。
- **US-302**：As 新手，I want 看到當下要做的動作 + 組數/次數，so that 不用記。
- **US-303**：As 新手，I want 完成一組後快速記下實際的重量和次數，so that 紀錄真實。
- **US-304**：As 新手，I want 一組結束後自動倒數休息時間，so that 不用看碼錶。
- **US-305**：As 新手，I want 暫停 / 取消 / 標記訓練結束，so that 突發狀況有彈性。
- **US-306**：As 新手，I want 訓練完成後看到本次摘要 (時長 / 噸位 / 動作數)，so that 有成就感。
- **US-307**：As 新手，I want 即使網路斷掉訓練也能繼續，so that 訓練不被打斷。

### Epic 4: 看回顧

- **US-401**：As 新手，I want 看到過去訓練的列表 (依日期)，so that 知道練了什麼。
- **US-402**：As 新手，I want 點入單次訓練看當時的所有組數，so that 對比上次表現。

### Epic 5: 客製化

- **US-501**：As 新手 (但好奇)，I want 自己組一套課表，so that 加入我喜歡的動作。
- **US-502**：As 新手，I want 編輯預設課表 (例如改休息秒數)，so that 符合我的習慣。
- **US-503**：As 新手，I want 切換 kg / lb 單位，so that 配合我的健身房。
- **US-504**：As 新手，I want 匯出我的訓練資料，so that 換手機不丟失。

### Epic 6: 「沒做、但要 say no 清楚」

明確不寫的 story：
- ❌ As 用戶，I want 加好友... → V1 無社群
- ❌ As 用戶，I want AI 幫我排課表... → V2
- ❌ As 用戶，I want 拍照記錄體態... → V2
- ❌ As 用戶，I want 記錄餐點熱量... → V2

## 7. 成功指標 (Success Metrics)

### 7.1 技術完成度 (Hard requirements)
- [ ] Lighthouse Performance ≥ 90、PWA ≥ 90、Accessibility ≥ 90
- [ ] E2E 主流程通過 (onboarding → 開始訓練 → 完成一組 → 結束訓練 → 看回顧)
- [ ] 完整離線狀態下可完成一次訓練 (含 Lottie 動畫播放)
- [ ] 業務邏輯單元測試覆蓋 ≥ 80% (`packages/core`)

### 7.2 用戶體驗 (UX targets — 自測)
- [ ] 新訪客 → 完成 onboarding → 開始訓練：總點擊 ≤ 6 次、總時間 ≤ 3 分鐘
- [ ] 訓練中：紀錄一組 ≤ 2 點擊
- [ ] iPhone Safari 安裝引導出現 → 安裝成功率 (自測) ≥ 70%

### 7.3 作品集價值 (Portfolio value)
- [ ] 公開 GitHub repo + 詳細 README
- [ ] Vercel 部署網址公開可訪問
- [ ] 至少 3 張關鍵架構圖在 README 顯示
- [ ] 30 秒 demo 影片 (錄屏 + 旁白)

### 7.4 V2 演進準備 (Architectural readiness)
- [ ] `packages/core` 可在 Node 環境跑測試 (證明無 browser dependency)
- [ ] RxDB schema 可直接對應 V2 Supabase Postgres schema (1:1 mapping 或文件記錄差異)
- [ ] AI Service Layer 介面已定義 (`AIPort`，見 [10-ai-extension-points.md](./10-ai-extension-points.md))

## 8. 約束條件 (Constraints)

| 約束           | 描述                                                                                     |
| -------------- | ---------------------------------------------------------------------------------------- |
| 預算           | $0 (個人專案、用免費額度)                                                                |
| 開發人數       | 1 人 (使用者)                                                                            |
| 時程           | 由使用者決定，但 SDD 假設「3 個月內可完成 V1」是合理目標                                  |
| 技術           | Web only (V1)、繁中為主、現代瀏覽器 (Chrome / Safari / Edge 近兩年版本)                  |
| 設計資源       | 透過 Claude Design 產出，不雇用設計師                                                    |
| 內容資源       | 動作說明參考公開資料 + 自寫，Lottie 素材由使用者自找 / 自製 / Claude Design 協助           |

## 9. 假設 (Assumptions)

> 這些是「相信為真、但未經驗證」的前提。實作中若發現假設不成立，SDD 需修正。

- A1: 健身新手願意花 3 分鐘做 onboarding 換取適配的內容
- A2: PWA 在 iOS Safari 上的「加到主畫面」流程，新手能在引導下完成
- A3: Lottie 動畫對新手而言、比靜態圖示有顯著教學提升 (否則改用 GIF 也 OK)
- A4: 預設 3 套課表足以涵蓋 80% 新手需求 (數字憑公開健身書經驗、非統計)
- A5: 用戶可接受「無雲端、換手機需手動匯出匯入」的 trade-off
